摸鱼小说阅读器 — 开发流程与后续步骤
=====================================
技术栈：Electron 35 + Vue 3 + TypeScript（纯本地，无后端，免 Visual Studio）
文档版本：v0.3.0  |  更新：2026-06-30
Git tag：v0.1.0（便携版初版）→ v0.3.0（本里程碑，见第十节）


一、项目目标（核心需求摘要）
--------------------------

1. 窗口可随意拖拽、调整大小（无最小尺寸限制）；文字随窗口实时重排
2. 窗口永远置顶；全局快捷键（可录制）快速隐藏/显示
3. 支持导入多种本地小说格式，多本书籍管理（UTF-8 / GBK 自动/手动）
4. 字号、文字色、背景色可自由调节；支持透明窗口摸鱼模式（真透视桌面）
5. 滚轮 + 方向键 + 自定义键 + 空格翻页
6. 禁止半行/半字截断；可选翻页保留上一页 n 行
7. 强记忆：阅读进度、窗口位置大小、样式设置、书签与章节

技术要求：
- Electron + Vue 3 + TypeScript
- 分页：ResizeObserver + DOM 实测折行（与屏幕渲染一致）
- 持久化：electron-store（书架 + 设置 + 窗口 bounds）
- 全局快捷键：Electron globalShortcut + ShortcutRecorder / PageKeyRecorder
- 文件：dialog.showOpenDialog + fileReader.cjs（iconv-lite / mammoth / jszip 等）
- Windows 无边框透明：winFrameless.cjs + dwmCaption.cjs（koffi 调 DWM）

三、v0.3.0 已完成功能总结（按模块）
----------------------------------

【架构 / 多窗口】
  - 主窗口仅阅读区（#reader）；设置 / 书架 / 导航为独立子窗口
    · electron/childWindows.cjs
    · SettingsWindowApp / ShelfWindowApp / NavigatorWindowApp（hash 路由）
  - 删除 TitleBar.vue；阅读时悬停顶栏 ReaderToolbar.vue（overlay）
  - 顶部 8px→40px 拖拽区；阅读区 -webkit-app-region: drag
  - IPC：open-settings/shelf/navigator-window、shelf-open-book、navigator-jump-offset
  - app-state-updated 广播，子窗口与主窗口状态同步

【导入与格式】electron/fileReader.cjs
  - 扩展名：txt, md, html, rtf, fb2, epub, docx, doc
  - 文本类：iconv-lite（GBK 回退）、HTML/RTF/FB2/EPUB（jszip）解析
  - Word：mammoth（docx）、word-extractor（doc）
  - normalizeBookText：可选合并空行（collapseBlankLines）
  - fixtures + fileReader.test.cjs（21 项）

【阅读与排版】
  - 翻页保留上一页 n 行：carryOverLinesEnabled / carryOverLines（pagination.ts）
  - 分页 LRU 缓存：pageCache.ts
  - 多显示器 DPI：display-metrics-changed → relayout
  - 透明文字：transparentText + textOpacity
  - 章节识别：utils/chapters.ts；书签 Ctrl+B

【设置与 UI】
  - 设置面板：主题预设、编码、空行合并、翻页键录制、进度滑块
  - 打开设置：顶栏 ⚙ / 右键阅读区 / Ctrl+,
  - 书架：排序、搜索、独立窗口
  - 导航：章节列表 + 书签跳转

【窗口与摸鱼（Windows 专项）】
  - 真透明：transparent: true + 窗口背景 #00000000（不用灰底/mica 妥协）
  - electron/winFrameless.cjs：无边框策略、空 title、WM_INITMENU 防菜单
  - electron/dwmCaption.cjs：DwmSetWindowAttribute 压制非客户区；
    剥离 WS_CAPTION/SYSMENU；ExtendFrameIntoClientArea；聚焦强制重绘
  - 不使用 titleBarOverlay（避免失焦顶栏 + 右上角系统蓝键）
  - focus-main-window IPC：点击正文恢复聚焦 + restoreWindowsFrameOnFocus
  - WM_ACTIVATE 钩子：激活时刷新 DWM 顶栏残留
  - 依赖：koffi（Windows 原生 API）

【托盘与快捷键】
  - 系统托盘 tray-icon.png；显示/隐藏/退出
  - 全局显隐快捷键可录制；翻页键可自定义
  - shortcutConflict：全局快捷键探测 + 翻页键冲突校验

【质量保障】
  - npm run test = vitest(32) + shortcut(2) + fileReader(21)
                  + dwmCaption(2) + winFrameless(5) = 62 项
  - scripts/create_test_docx.py / create_test_epub.py（测试素材）

五、已知限制与平台说明（重评估）
------------------------------

【Windows 失焦顶栏】
  - 根因：DWM 对 frameless + transparent 窗口在失焦时可能绘制非客户区装饰
  - 已尝试：禁用 NC 渲染、COLOR_NONE、剥 caption 样式、WM_ACTIVATE 重绘、
    点击正文 focus + forceRepaint；已移除 titleBarOverlay（解决右上角系统蓝块）
  - 限制：部分 Win10/11 构建上仍可能偶发顶栏闪烁；属 Electron/DWM 已知类问题
  - 不推荐回退：灰底/mica 假透明（牺牲真摸鱼透视）

【聚焦与拖拽】
  - 点击正文 → focusMainWindow + 全阅读区可 drag
  - 失焦后顶栏视觉恢复依赖 restoreWindowsFrameOnFocus（若仍异常需报 Win 版本）

【macOS / Linux】
  - 主流程可用；winFrameless/dwmCaption 仅 win32 生效
  - 未做专项透明顶栏测试

【架构选择】
  - 保留 Electron + Vue（非原生 WPF）；阅读区为 WebView 渲染，窗口层做平台差异化

七、日常开发工作流（强制顺序）
------------------------------

  1. 确认阶段编号（对照第四节、第六节）
  2. 改代码
  3. npm run test          ← 62 项必须通过
  4. npm run electron:dev  ← 手动验证
  5. 自测清单：
       【阅读】导入 txt/md/epub/docx；极小/极大窗口；无半行截断；续行 n 行
       【窗口】拖拽（顶栏+正文）；置顶；全局显隐快捷键
       【透明】0% 透视桌面；失焦→点正文：任务栏聚焦 + 顶栏视觉恢复
       【子窗口】设置/书架/导航打开关闭；章节跳转；书签 Ctrl+B
       【持久化】重启后进度、窗口位置、样式恢复
  6. npm run build
  7. （发版时）npm run dist
  8. git commit → tag → push
  9. 更新本文件阶段状态
