摸鱼小说阅读器 — 开发流程与后续步骤
=====================================
技术栈：Electron + Vue 3 + TypeScript（纯本地，无后端，免 Visual Studio）
文档版本：v0.1.0  |  更新：2026-06-30


一、项目目标（核心需求摘要）
--------------------------

1. 窗口可随意拖拽、调整大小（无最小尺寸限制）；文字随窗口实时重排
2. 窗口永远置顶；全局快捷键（可录制，如 Ctrl+Z）快速隐藏/显示
3. 支持导入 .txt，多本书籍管理（UTF-8 / GBK 自动识别）
4. 字号、文字色、背景色可自由调节；支持透明窗口摸鱼模式
5. 滚轮 + 方向键 + J/K + 空格翻页
6. 禁止半行/半字截断
7. 强记忆：阅读进度、窗口位置大小、样式设置

技术要求：
- Electron + Vue 3 + TypeScript
- 分页：ResizeObserver + DOM 实测折行（与屏幕渲染一致）
- 持久化：electron-store（书架 + 设置 + 窗口 bounds）
- 全局快捷键：Electron globalShortcut + ShortcutRecorder 按键录制
- 文件：dialog.showOpenDialog + fs（iconv-lite GBK 回退）


二、目前已完成变动总结（v0.0 → v0.1.1）
----------------------------------------

【架构】
  - 由 Tauri 迁移至 Electron（公司电脑无法装 MSVC / Visual Studio）
  - 弃用 src-tauri/，主进程在 electron/main.cjs
  - 全部数据本地存储，不连接任何后端

【窗口与摸鱼体验】
  - 无边框、透明窗口、可置顶
  - 阅读时自动隐藏标题栏，仅保留文字与背景
  - 鼠标移至顶部 / 右键 / Ctrl+S 唤出设置
  - 透明窗口模式 + 背景不透明度滑块
  - 窗口最小尺寸限制已移除（可缩到任意大小）

【导入与持久化】
  - 修复 IPC「无法被克隆」：Vue 响应式对象转 plain JSON 再传输
  - 修复文件对话框被置顶窗口遮挡
  - GBK / UTF-8 智能读取
  - 书架、阅读进度、窗口位置自动保存

【快捷键】
  - ShortcutRecorder：点击录制，Ctrl+Z 等自动识别并注册
  - 注册失败时设置面板显示错误提示

【排版】
  - Canvas 测量改为 DOM 实测，消除下半屏空白
  - 去掉末尾空行填充；行高 DOM 探测
  - 字号改为数字输入，最小 1px，无 12px 下限

【质量保障】
  - Vitest 单元测试（shortcut / pagination / color / serialize 等）
  - electron 主进程 node:test（shortcut / fileReader）
  - 每次改动流程：npm run test → electron:dev 手测 → npm run build

五、日常开发工作流（强制顺序）
------------------------------

  1. 确认阶段编号（对照第四节）
  2. 改代码
  3. npm run test          ← 单元测试必须通过
  4. npm run electron:dev  ← 手动验证
  5. 自测清单：
       - 导入 TXT 正常？
       - 极小/极大窗口排版正常、无半行截断？
       - 透明模式、标题栏隐藏、快捷键录制？
       - 重启后进度/窗口/样式恢复？
  6. npm run build
  7. git commit → push
  8. 更新本文件阶段状态
