摸鱼小说阅读器 — 开发流程与后续步骤
=====================================
技术栈：Electron 35 + Vue 3 + TypeScript（纯本地，无后端，免 Visual Studio）
文档版本：v0.6.0-release  |  更新：2026-07-01
Git tag：v0.1.0 → v0.3.0 → v0.4.0 → v0.5.0 → v0.6.0（已发布）


一、项目目标（核心需求摘要）
--------------------------

1. 窗口可随意拖拽、调整大小（无最小尺寸限制）；文字随窗口实时重排
2. 窗口永远置顶；全局快捷键（可录制）快速隐藏/显示
3. 支持导入多种本地小说格式，多本书籍管理（UTF-8 / GBK 自动/手动）
4. 字号、文字色、背景色可自由调节；支持透明窗口摸鱼模式（真透视桌面）
5. 滚轮 + 方向键 + 自定义键 + 空格翻页
6. 禁止半行/半字截断；可选翻页保留上一页 n 行
7. 强记忆：阅读进度、窗口位置大小、样式设置、书签与章节

技术要求：
- Electron + Vue 3 + TypeScript
- 分页：ResizeObserver + DOM 实测折行（与屏幕渲染一致）
- 持久化：electron-store（书架 + 设置 + 窗口 bounds）
- 全局快捷键：Electron globalShortcut + ShortcutRecorder / PageKeyRecorder
- 文件：dialog.showOpenDialog + fileReader.cjs（iconv-lite / mammoth / jszip 等）
- Windows 无边框透明：winFrameless.cjs + dwmCaption.cjs（koffi 调 DWM）

五、已知限制与平台说明（重评估）
------------------------------

【Windows 失焦顶栏】
  - 根因：DWM 对 frameless + transparent 窗口在失焦时可能绘制非客户区装饰
  - 已尝试：禁用 NC 渲染、COLOR_NONE、剥 caption 样式、WM_ACTIVATE 重绘、
    点击正文 focus + forceRepaint；已移除 titleBarOverlay（解决右上角系统蓝块）
  - 限制：部分 Win10/11 构建上仍可能偶发顶栏闪烁；属 Electron/DWM 已知类问题
  - 不推荐回退：灰底/mica 假透明（牺牲真摸鱼透视）

【聚焦与拖拽】
  - 阅读区默认 no-drag，滚轮可直接翻页
  - 按住 Alt → 阅读区变为拖动手柄；松开后恢复尺寸
  - 顶栏左侧 window-drag-strip 也可拖窗口
  - 失焦后顶栏视觉恢复依赖 restoreWindowsFrameOnFocus（若仍异常需报 Win 版本）

【macOS / Linux】
  - 主流程可用；winFrameless/dwmCaption 仅 win32 生效
  - 未做专项透明顶栏测试

【架构选择】
  - 保留 Electron + Vue（非原生 WPF）；阅读区为 WebView 渲染，窗口层做平台差异化

（一）需求 A：透明背景下文字颜色随桌面明暗自动切换
------------------------------------------------

【用户期望】
  开启透明窗口后，透视到的桌面偏黑 → 文字自动变白；
  桌面偏白 → 文字自动变黑。

【结论】可实现，建议作为「自适应对比度」功能；需设合理预期，非 100% 像素级精确。

【技术可行性】★★★★☆（中高）

  能做到：
  · 在透明摸鱼模式开启时，周期性估算窗口后方屏幕区域的平均亮度
  · 亮度低于阈值 → 使用浅色字（如 #f5f5f5）；高于阈值 → 深色字（如 #1a1a1a）
  · 窗口移动、换壁纸、切换显示器后可重新采样（可防抖，避免闪烁）

  难点与限制（必须在说明书中写明）：
  · Electron 渲染层无法直接「读取透明像素后面的桌面」，必须靠主进程截屏采样
  · 窗口置顶且覆盖桌面时，截屏需裁切窗口所在显示器区域；采样的是「该区域
    当前画面」，若后方是彩色壁纸/图标/视频，会得到混合色而非纯黑/纯白
  · 周期性截屏有性能开销（建议 1.5～3 秒一次，仅在 autoContrast 开启时）
  · 阅读区若使用不透明背景色/高不透明度，后方桌面影响变小，自适应效果减弱
  · 多显示器、HDR、远程桌面等场景需额外测试

【推荐实现路径（分两步）】

  Step A — MVP（v0.4.0 必做）
  ─────────────────────────
  1. 设置项 ReaderSettings 扩展：
       autoTextContrast: boolean        // 开关，默认 false
       autoTextContrastThreshold: number // 亮度阈值 0～255，默认 128
  2. 主进程 electron/desktopLuminance.cjs（或同类模块）：
       · 使用 desktopCapturer / 显示器区域截图（npm 备选：node-screenshots）
       · 根据 mainWindow.getBounds() 取窗口中心附近 N×N 像素网格
       · 计算相对亮度：Y = 0.299R + 0.587G + 0.114B
       · 返回 { luminance, suggestLightText: boolean }
  3. IPC：sample-desktop-luminance → 渲染进程
  4. 触发时机：
       · transparentBackground === true 且 autoTextContrast === true
       · 窗口 show / move / resize / display-metrics-changed
       · 定时器兜底（2s，窗口可见且未 hide 时）
  5. 渲染进程 utils/color.ts 增加 resolveAutoTextColor(luminance) 
     ReaderView 在自动模式下覆盖 settings.textColor（不写入持久化手动色）

  计时规则（建议写入产品说明）：
  · 仅统计【主阅读窗口可见】时长（isVisible() === true）
  · hide() / 全局快捷键隐藏 / 托盘「隐藏窗口」→ 暂停计时
  · show() / 快捷键显示 / 托盘「显示窗口」→ 继续计时
  · 子窗口（设置/书架/导航）打开不影响计时（主窗口仍可见）
  · 系统休眠 powerMonitor('suspend') 暂停；resume 时若仍可见则继续
  · 退出应用前 flush 当前会话到累计值

【数据模型扩展】

  // AppState 顶层或 settings 旁
  interface MoyuStats {
    totalVisibleMs: number;      // 历史累计摸鱼毫秒
    lastSessionStartAt: number | null; // 仅运行时，可不持久化
  }

  // ReaderSettings 或独立 MoyuSalarySettings
  interface MoyuSalarySettings {
    salaryEnabled: boolean;           // 是否显示金额
    salaryMonthly: number;            // 月薪（元），默认 0
    salaryWorkDaysPerMonth: number;   // 每月工作天数，默认 22
    salaryHoursPerDay: number;        // 每日工时，默认 8
  }

  金额公式：
    时薪 = salaryMonthly / (salaryWorkDaysPerMonth × salaryHoursPerDay)
    等价金额 = (totalVisibleMs / 3_600_000) × 时薪
    本次金额 = (currentSessionMs / 3_600_000) × 时薪

  R6  翻页伴随全量存盘 IPC                → 每次翻页 updateBookProgress，仍有延迟
