摸鱼小说阅读器 — 开发流程与后续步骤
=====================================
基于需求：Tauri v2 + Vue 3 + TypeScript 浮动 TXT 阅读器
文档版本：MVP v0.1  |  更新：2026-06-29


一、项目目标（核心需求摘要）
--------------------------

1. 窗口可随意拖拽、调整大小；文字显示数量随窗口实时变化（动态分页）
2. 窗口永远置顶；支持点击特定区域或全局快捷键（可配置，如 Ctrl+` / F12）快速隐藏/显示
3. 支持导入 .txt 文件，多本书籍管理
4. 可实时调整字体大小、背景颜色（护眼模式）、文字颜色
5. 支持鼠标滚轮 + 键盘（方向键、J/K、空格等）翻页，快捷键可自定义
6. 绝对不能出现半截字/半行被切断，必须完整显示文字内容
7. 强记忆能力：关闭后恢复阅读位置（按书籍 char offset）、窗口位置大小、字体/颜色设置

技术要求：
- Tauri v2 + Vue 3 + TypeScript
- 分页：ResizeObserver + canvas measureText / line-height 精确计算
- 进度持久化：Tauri Store / JSON（per book）
- 窗口状态：tauri-plugin-window-state
- 全局快捷键：@tauri-apps/plugin-global-shortcut
- 文件导入：tauri-plugin-dialog + fs

三、各阶段详细流程
------------------

【阶段 0】环境准备与仓库初始化  ✅ 已完成
  0.1 克隆/初始化 Git 仓库
  0.2 整理工作区目录（避免嵌套 worktree 同名冲突）
  0.3 安装 Node.js 18+、Rust 1.77+
  0.4 安装 Windows 构建依赖：Visual Studio Build Tools（含 C++ 桌面开发，提供 link.exe）
  0.5 验证：npm install、npm run build（前端）、npm run tauri dev（完整应用）

【阶段 1】项目脚手架 + 目录结构 + 数据模型  ✅ 已完成
  1.1 初始化 Tauri v2 + Vue 3 + TypeScript 项目
  1.2 配置插件：window-state、global-shortcut、dialog、fs、store
  1.3 定义目录结构：
        src/types/          类型定义
        src/services/       业务服务（storage / pagination / bookImport）
        src/composables/    可复用逻辑（窗口 / 快捷键）
        src/components/     UI 组件
        src-tauri/          Rust 后端与 capabilities
  1.4 定义数据模型：
        Book           书籍元数据 + charOffset 阅读进度
        ReaderSettings 字体 / 颜色 / 置顶 / 隐藏快捷键
        AppState       书架列表 + 设置 + lastBookId

【阶段 2】基础窗口能力  ✅ 已完成
  2.1 tauri.conf.json：无边框、可缩放、初始隐藏、alwaysOnTop
  2.2 TitleBar：data-tauri-drag-region 拖拽区域
  2.3 置顶：setAlwaysOnTop，设置面板可切换
  2.4 隐藏/显示：标题栏按钮 + 全局快捷键 toggleVisibility
  2.5 窗口状态记忆：tauri-plugin-window-state 自动保存/恢复位置与大小

【阶段 3】阅读视图 + 动态分页核心  ✅ 已完成
  3.1 ReaderView 阅读容器 + ResizeObserver 监听尺寸变化
  3.2 pagination.ts：canvas.measureText 逐字/逐词折行
  3.3 按完整行数计算每页（maxLines = floor(高度 / lineHeight)）
  3.4 以 charOffset 记录页首位置，翻页更新 offset，禁止半行截断
  3.5 窗口 resize 时从当前 offset 重新分页渲染

【阶段 4】TXT 导入 + 翻页 + 设置面板（MVP 闭环）  ✅ 已完成

  4.2 书架：多书管理、切换、删除
  4.3 翻页：滚轮、方向键、J/K、空格
  4.4 设置面板：字号、文字色、背景色、置顶、隐藏快捷键
  4.5 Store 持久化：阅读进度、样式、lastBookId；关闭重开自动恢复

四、日常开发工作流（每次改动的标准顺序）
----------------------------------------

  1. 从 main 拉取最新代码（或确认在正确 worktree 分支上）
  2. 明确本次属于哪个阶段 / 哪个子任务（对照上文编号）
  3. 改代码 → 本地 npm run tauri dev 热重载验证
  4. 自测清单：
       - 分页是否截断？
       - 重启后进度/窗口/样式是否恢复？
       - 隐藏快捷键是否仍可用？
  5. npm run build 确认前端 TypeScript 无报错
  6. git add → commit（附清晰说明）→ push
  7. 若完成一个阶段节点，更新本文件中的阶段状态（✅ / ⬅ / ⬜）
